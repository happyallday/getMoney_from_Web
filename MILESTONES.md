## 提交规范

### 分支策略
- `main`: 主分支，每个里程碑完成后合并
- `develop`: 开发分支
- `feature/*`: 功能分支
- `bugfix/*`: bug修复分支

### 提交信息格式
```
milestoneX-子任务编号: 简短描述

详细描述（如果需要）
```

例如:
```
milestone1-1: 创建项目目录结构和基础文档
```

### 标签管理
每个里程碑完成后创建相应的GitHub Release和Tag：
```bash
git tag -a v0.1.0 -m "Milestone 1: 项目初始化"
git push origin main --tags
```
